Week 1: Security-First Foundations Beginner
â–¼
1 Secure Linux Hardening Lab
Set up a VM and harden it using CIS benchmarks. Configure SSH key auth, disable root login, setup fail2ban, and implement basic firewall rules with UFW.
Linux SSH UFW
2 Network Security Scanning
Use Nmap to scan your network. Identify open ports, running services, and potential vulnerabilities. Document findings and create a remediation plan.
Networking Nmap
3 Docker Security Basics
Build a Docker image following security best practices: non-root user, minimal base image, no secrets in layers. Scan with Trivy for vulnerabilities.
Docker Trivy
4 Git Secrets Detection
Set up git-secrets or TruffleHog. Scan a repository for exposed credentials, API keys, and passwords. Create pre-commit hooks to prevent future leaks.
Git Security
5 AWS IAM Security Audit
Review IAM policies in your AWS account. Identify overly permissive roles, unused access keys, and implement least-privilege principles. Set up MFA.
AWS IAM
Week 2: CI/CD Security Pipeline Intermediate
â–¼
6 Build a Secure CI/CD Pipeline
Create a GitHub Actions pipeline that includes SAST scanning (with Semgrep or SonarQube), dependency checking, and automated security tests.
CI/CD SAST GitHub Actions
7 Container Registry Security
Set up a private container registry with authentication. Implement image signing with Cosign and create policies to only allow signed images.
Docker Cosign
8 Infrastructure as Code Security
Write Terraform code to provision AWS resources. Use Checkov or tfsec to scan for misconfigurations before deployment.
Terraform AWS Checkov
9 Ansible Security Playbooks
Create Ansible playbooks to automate security configurations: install security updates, configure logging, setup intrusion detection with AIDE.
Ansible Automation
10 Secrets Management
Deploy HashiCorp Vault. Store database credentials, API keys, and certificates. Integrate with your applications to retrieve secrets dynamically.
Vault Secrets
Week 3: Kubernetes Security Mastery Intermediate
â–¼
11 K8s RBAC Implementation
Set up Role-Based Access Control in your Kubernetes cluster. Create custom roles with minimal permissions for different teams and services.
Kubernetes RBAC
12 Pod Security Standards
Implement Pod Security Policies/Admission Controllers. Prevent privileged containers, enforce read-only root filesystems, and drop capabilities.
Kubernetes Security
13 Network Policies Lab
Create Kubernetes Network Policies to control traffic between pods. Implement zero-trust networking with Calico or Cilium.
Kubernetes Networking
14 Runtime Security Monitoring
Deploy Falco for runtime threat detection. Create custom rules to detect suspicious activities like shell executions in containers.
Kubernetes Falco
15 K8s Security Audit
Use kube-bench and kube-hunter to audit your cluster against CIS benchmarks. Document and fix identified vulnerabilities.
Kubernetes Audit
